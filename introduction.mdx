The OSS Review Toolkit (ORT) is a FOSS policy automation and orchestration toolkit that helps you manage open source software dependencies in a strategic, safe, and efficient manner.

Use ORT to generate CycloneDX and SPDX SBOMs, automate license and vulnerability compliance checks, and produce attribution notices — all from a single, configurable pipeline.

## The ORT pipeline

ORT is built as a pipeline of composable tools. Each tool produces an `OrtResult` file (YAML or JSON) that the next tool consumes.

<Steps>
  <Step title="Analyze">
    The **Analyzer** detects your project's package manager and resolves all dependencies into a full dependency graph.
  </Step>
  <Step title="Scan (optional)">
    The **Scanner** downloads source code and runs license/copyright scanners (ScanCode, FossID, Licensee, etc.) on it.
  </Step>
  <Step title="Advise (optional)">
    The **Advisor** queries vulnerability databases (OSV, OSS Index, VulnerableCode) for security advisories on your dependencies.
  </Step>
  <Step title="Evaluate">
    The **Evaluator** applies your custom policy rules (written in Kotlin) to flag license violations or other compliance issues.
  </Step>
  <Step title="Report">
    The **Reporter** generates output in your desired format: CycloneDX SBOM, SPDX SBOM, static HTML, web app, plain text notices, and more.
  </Step>
</Steps>

## Quick example

```bash
# Analyze a project's dependencies
ort analyze -i /path/to/project -o /path/to/output

# Scan source code for licenses
ort scan -i /path/to/output/analyzer-result.yml -o /path/to/scan-output

# Generate a CycloneDX SBOM
ort report -f CycloneDX -i /path/to/scan-output/scan-result.yml -o /path/to/reports
```

<Note>
  ORT requires Java 21 or later. Run `ort requirements` to see which external tools are needed for your project's package managers.
</Note>
